When designing a large system, several key factors must be considered:

- What are the various architectural components that can be used?
- How do these components interact with one another?
- How can we best utilize these components: what are the right tradeoffs?

Investing in scaling before it is required is generally not a wise business decision; however, incorporating forethought into system design can save valuable time and resources down the line. In the following chapters, we will define some of the core building blocks of scalable systems. Familiarity with these concepts will greatly benefit the understanding of distributed systems. In the upcoming sections, we will explore Consistent Hashing, CAP Theorem, Load Balancing, Caching, Data Partitioning, Indexes, Proxies, Queues, Replication, and choosing between SQL vs. NoSQL.
